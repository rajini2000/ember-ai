## What Was Built

A Reinforcement Learning (DQN) agent that replaces the rule-based AQI alarm system.
Trained on synthetic data + real Beijing PM2.5/PM10 dataset.

## Datasets Used

### Beijing Multi-Site Air Quality (Kaggle)
- URL: https://www.kaggle.com/datasets/sid321axn/beijing-multisite-airquality-data-set
- 12 CSV files saved in `data/real/`
- Columns used: PM2.5, PM10, TEMP, PRES, DEWP, CO

### Synthetic Data
- Auto-generated by `data_generator.py`
- Based on sensor ranges in SYSTEM_DOCUMENTATION.md Section 9
- 300 clean air episodes + 200 pollution episodes
- Saved to `data/synthetic/synthetic_training_data.csv`

## How the AI Works

1. **State space** — 16 sensor features (PM1.0, PM2.5, PM10, TVOC, eCO2, temperature, humidity, pressure, gas resistance, MQ voltage, MQ digital, PMS valid, ENS160 available, BME680 available, AQI score, alarm state)
2. **Action space** — 2 actions: `0 = ALARM OFF`, `1 = ALARM ON`
3. **Reward function** (from SYSTEM_DOCUMENTATION.md Section 7.3):
   - Correct detection (danger + alarm on): +10
   - Missed danger (danger + alarm off): -50
   - Correct silence (safe + alarm off): +1
   - False alarm (safe + alarm on): -5
   - Fast response bonus: +15
   - Fast recovery bonus: +5

## When Mirac Sends Hardware CSV Files

1. Copy CSV files into `data/real/`
2. Retrain:
```bash
cd src
python train.py --hardware ../data/real/YYYY-MM-DD.csv
```
3. Evaluate:
```bash
python evaluate.py
```
4. Send Mirac the new `models/best_model.zip`

**Tell Mirac to collect:**
- Several hours of normal clean air data
- Data with vape/smoke near the sensor (real danger events)

```python
    # send Twilio SMS
```

## CSV Format (Hardware SD Card)

```
timestamp,PM1.0,PM2.5,PM10,TVOC,eCO2,temperature,humidity,pressure,gas,MQ_analog,MQ_digital
2026-02-22 14:30:15,8,15,18,0,0,27.3,18.2,990.9,14523678.0,0.031,1
```

- TVOC and eCO2 = 0 (ENS160 disconnected)
- gas = BME680 gas resistance in Ohms
- MQ_analog = ADC reading 0.0 to 1.0 (multiply by 3.3 for voltage)
